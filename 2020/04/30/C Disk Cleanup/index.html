C Disk Cleanup--Solution to the problem that the emulator (Yeshen as an example) takes up more and more space

solution clean clean 2020/04/30

#C Disk Cleanupâ€“Solution to the problem that the emulator (Yeshen as an example) takes up more and more space (this method is suitable for the situation where multiple APKs have been installed, although they are uninstalled, they are still occupied)

This method is suitable for the situation where multiple APKs have been installed, although they are uninstalled but the occupation is still serious

Happening:

At the beginning, I had 15G left on the C drive. After installing the APK, I reduced it to 5G. In order to make room for the PS, I uninstalled almost all the applications in the virtual machine. However, I found that the C drive did not become larger and the occupation was still serious. Find the location of the night god simulator and check that it takes up 20G of memory, plus other learning software forcibly occupied my 200G C drive.

Using the cleanup program that comes with the Night God Simulator is invalid.

idea:

I guess that the simulation method used by the Night God Simulator is similar to the Vm virtual machine. Find the corresponding folder, and you can see that the generated vmdk file is the same as the Vm virtual machine. It should be that the Night God Simulator is not perfect enough. The uninstall file on the interface is just not displayed on the interface, and the file is not actually deleted, and the Vm virtual machine does not have this problem.

Solution:

\1. First export the useful applications, press and hold the application to select export, after the export is completed, it will jump directly, or you can find the exported file in C:\Users\Administrator\Nox_share\ImageShare.

\2. Find the directory C:\Software\Applications\Emulator Night God\Nox\bin\BignoxVMS\nox

Delete the nox-disk2.vmdk file, if it shows occupied, close the Night God simulator and try again, or use the unoccupied tool, if it does not restart the computer.

\3. Reopen the virtual machine and reinstall the backed up files. You can see that the junk files are successfully removed.
